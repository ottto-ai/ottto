### ottto --help
Ottto local platform CLI for developers, support, CI, and AI agents

Usage: ottto [OPTIONS] <COMMAND>

Commands:
  status        Show daemon, account, relay, update, and app health
  apps          List apps or refresh app setup status
  agent-status  Refresh one source status using the lower-level source noun
  setup         Connect this Mac through a browser claim
  login         Sign in and connect this Mac through a browser claim
  account       Show the Ottto account connected to this Mac
  logout        Disconnect this Mac from Ottto
  doctor        Run daemon health checks and print current status
  fix           Apply daemon-approved repair for one app
  verify        Verify one app and publish safe setup status
  diagnostics   Collect local-only or approved support diagnostics
  update        Check owner-aware update state and instructions
  uninstall     Remove Ottto local runtime state for this user
  help          Print this message or the help of the given subcommand(s)

Options:
      --socket <SOCKET>  Override the ottto-service Unix socket path
      --token <TOKEN>    Override the local-control token for CLI and agent requests
      --no-autostart     Do not kickstart the standard per-user ottto-service
      --watch            Emit NDJSON progress events and a final event; requires --json
  -h, --help             Print help

### ottto setup --help

Options:
      --claim-code <CLAIM_CODE>  Browser claim code from the Ottto Apps page
      --socket <SOCKET>          Override the ottto-service Unix socket path
      --no-browser               Do not open the browser; print the claim URL and code
      --token <TOKEN>            Override the local-control token for CLI and agent requests
      --no-autostart             Do not kickstart the standard per-user ottto-service
      --no-wait                  Return after starting or attaching setup without waiting
      --timeout <SECONDS>        Seconds to wait for browser approval and setup progress [default: 300]
